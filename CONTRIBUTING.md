# Contributing to HttpForge

Thank you for your interest in contributing to HttpForge. This document covers how to get started with the project website and related repositories.

## Code of Conduct

This project follows our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold it.

## Ways to Contribute

### Code

1. Fork the repository on GitHub
2. Create a branch: `git checkout -b feature/your-feature`
3. Make your changes with clear, focused commits
4. Submit a pull request with a description of the change and why it is needed

For significant changes, open an issue or discussion first to align on the approach.

### Documentation

Improve guides, fix typos, add examples, or clarify existing content. Documentation changes follow the same pull request workflow.

### Bug Reports

Open a [GitHub issue](https://github.com/httpforge/issues) with:

- Steps to reproduce
- Expected vs. actual behavior
- Environment details (OS, app version, etc.)

### Feature Requests

Share ideas in [GitHub Discussions](https://github.com/httpforge/discussions). We prioritize features based on community demand and alignment with our [roadmap](https://httpforge.com/roadmap.html).

## Pull Request Guidelines

- Keep changes focused and scoped to a single concern
- Match existing code and documentation style
- Update relevant docs when behavior changes
- Describe what changed and why in the PR description

## Security Issues

Do not report security vulnerabilities through public issues. See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.
